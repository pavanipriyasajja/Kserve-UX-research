## What is GitHub repository Mining in UX Research?

GitHub mining or Github repository Mining is a UX research method that systematically analyzes GitHub issues, discussions, pull requests, feature requests, and documentation feedback to identify recurring usability problems, developer pain points, workflow challenges, and opportunities for improving the developer experience. Unlike traditional user research methods that rely on interviews or surveys, GitHub mining leverages real-world data generated by developers while using a product in production environments.

## Why Did We Choose This Research Method?

We decided to use GitHub mining as a complementary research method for several reasons:

<b>Global open-source communities operate across multiple time zones.</b> Coordinating interviews with users, contributors, and maintainers was challenging due to differences in availability and geographical distribution, making it difficult to recruit participants consistently.

<b>Limited availability of engineers and maintainers.</b> Many contributors volunteer their time outside of their regular work, which reduced opportunities for conducting a large number of synchronous usability sessions.

<b>Findings from our usability studies highlighted support challenges.</b> During our usability interviews, several engineers shared that they were often unable to get timely or sufficient help through GitHub issues. This indicated an opportunity to investigate GitHub interactions more closely to understand where communication, documentation, or issue resolution was breaking down.

<b>Access to a larger and more diverse dataset.</b> GitHub issues represent experiences from developers with different roles, Kubernetes environments, cloud providers, deployment scenarios, and KServe versions, providing a broader perspective than interviews alone.

<b>Evidence-based insights from real-world usage.</b> Rather than relying solely on participants' recollections, GitHub mining captures actual problems encountered during deployment, configuration, debugging, observability, upgrades, and documentation, allowing us to identify recurring patterns supported by real engineering discussions.

Initially, we planned to conduct usability studies to better understand developers' experiences with KServe (Here is the project proposal document and related document link (https://github.com/kserve/kserve/issues/4979 ). 

Although <b>13 engineers expressed interest in participating, only one engineer completed a usability study interview. The limited participation was primarily due to the global distribution of users and maintainers across different time zones, scheduling challenges, and limited availability.</b> Based on these constraints, we decided to shift our research toward GitHub mining. This approach allowed us to analyze a much larger set of real-world developer experiences captured in GitHub issues, providing scalable, evidence-based insights into common usability challenges, documentation gaps, deployment issues, and developer workflows. By leveraging GitHub mining, we were able to continue the research despite the limitations of recruiting participants and generate actionable recommendations for KServe maintainers to improve the overall developer experience.
